# 엔드필드 공장 모듈 배치 도구 — 프로젝트 스펙 (CLAUDE.md)

> 이 문서는 Claude Code(Code 탭)가 프로젝트 폴더를 열 때 자동으로 읽는 컨텍스트 파일입니다.
> 목적: 명일방주: 엔드필드(Arknights: Endfield)의 통합 공업(AIC) 설비를 자유롭게 배치·연결해볼 수 있는
> 드래그 앤 드롭 기반 "공장 레이아웃 플래너" 웹 도구를 만든다.
> 조사 기준일: 2026-07-19, 게임 버전 1.4 「심연으로」 기준.

## 1. 프로젝트 목표

- 사용자가 그리드 위에 공업 설비(채광기, 정제실, 분쇄실, 조립실, 포장, 전력, 저장 등)를 자유 배치
- 설비 간 컨베이어 벨트/파이프 연결을 시각적으로 그릴 수 있어야 함
- 전력망(PAC → 중계탑 → 철탑) 커버리지를 시각적으로 확인할 수 있으면 좋음 (범위 원으로 표시)
- 배치한 레이아웃을 저장/불러오기(JSON export/import) 가능해야 함
- 실제 게임처럼 "정확한 좌표/판정"까지 재현할 필요는 없음 — 플레이어가 아이디어를 스케치하고
  생산 체인이 병목 없이 맞물리는지 미리 검증하는 용도의 **기획 도구**에 가까움

## 2. 도메인 모델 (제안)

```
Module (설비 인스턴스)
  - id
  - facilityTypeId  → FacilityType 참조
  - position {x, y}  (그리드 좌표)
  - rotation (0/90/180/270)
  - ioPorts: [{ side: 'N'|'E'|'S'|'W', kind: 'input'|'output', resource: string }]

FacilityType (설비 정의 — 데이터로 분리, 하드코딩 금지)
  - id, name, category
  - footprint {w, h}   // 그리드 상 차지 크기 (기본 1x1, 큰 설비는 조정 가능하게)
  - powerDraw (kW 등, 상대값이라도 OK)
  - inputs: [{resource, rate}]
  - outputs: [{resource, rate}]
  - iconColor / category color

Connection (벨트/파이프/전선)
  - id, fromModuleId, fromPort, toModuleId, toPort, path(옵션, 자유곡선 or 그리드 경로)

PowerNode (PAC/중계탑/철탑)
  - id, type: 'PAC'|'relay'|'pylon', position, range(예: 중계탑 80m)
```

**중요:** 실제 게임 내 정확한 그리드 셀 크기, 설비 픽셀/미터 단위 footprint, 정확한 산출 속도(초당 개수)는
공식적으로 문서화되어 있지 않고 커뮤니티마다 표기가 다릅니다. `FacilityType` 데이터는 하드코딩하지 말고
`data/facilities.json` 같은 별도 파일로 분리해서, 사용자가 직접 값을 보정/추가할 수 있게 설계하세요.

## 4. 참고용 생산 체인 예시 (레시피 시드 데이터로 활용)

- **LC 계곡 배터리**: 원소금 분쇄 → 자수정 정제 → 부품 조립 → 포장
- **SC 계곡 배터리**: 철스러움 기반, LC보다 고출력
- **산업용 폭발물**: 자수정 정제 → 에코틴 씨앗 분쇄 → 포장
- **식양 (v1.4 기체공업 이전)**: 카본 조각 3단계 가공 → 청정수와 반응
- **식양 (v1.4 기체공업 이후)**: 비활성 기체 환경에서 카본 조각 + 청정수 → 직접 합성 (공정 단축 사례로 UI에서
  "기존 방식 vs 기체공업 방식" 비교 토글을 만들면 재미있을 듯)
- **중식양**: 기체 정제로 식양에서 직접 생산 (천화로/양정 폐액 불필요)

## 5. 전력/연결 규칙 요약

- PAC(메인/서브) → 중계탑(80m 범위 체인) → 전기 철탑 순으로 무선 전력 전달
- 설비는 전력 범위 안에 있어야 가동. 플래너에서는 중계탑/철탑을 배치하면 반경 원을 그려서
  커버리지를 시각화하는 기능을 넣으면 실용적임
- v1.3부터 "예비 전원 시스템" 존재 → 완전 정전 방지 (플래너에서는 굳이 시뮬레이션할 필요 없음, 참고만)

## 6. MVP 기능 범위 제안 (우선순위 순)

1. 그리드 캔버스 + 설비 팔레트에서 드래그 앤 드롭 배치, 회전
2. 설비 클릭 시 인풋/아웃풋 정보 툴팁 표시
3. 벨트로 두 설비의 포트 연결 (클릭-클릭 또는 드래그)
4. 레이아웃 JSON export/import (블루프린트 저장/불러오기)
5. 전력 커버리지 반경 시각화 (선택 기능)
6. 생산 체인 검증: 인풋 소비 속도 vs 아웃풋 생산 속도 비율 경고 표시 (병목 하이라이트)
7. (선택) 실제 게임 블루프린트 코드 형식과 별개로, 이 도구만의 공유 가능한 코드/URL 생성

## 7. 기술 스택 제안

- 프레임워크 제약 없음 — 빠르게 만들 거면 **Vanilla JS + HTML5 Canvas(또는 SVG)** 단일 파일로도 충분
- 좀 더 유지보수하기 좋게 하려면 **React + react-flow 또는 Konva.js** (노드 기반 에디터에 적합)
- 데이터는 `data/facilities.json`, `data/recipes.json`으로 분리해 나중에 패치 반영 시 코드 수정 없이 값만 갱신
- 상태 저장은 로컬(localStorage 또는 파일 export)로 충분, 백엔드 불필요

## 8. 데이터 정확도에 대한 주의사항

- 이 문서의 설비/체인 정보는 커뮤니티 가이드(나무위키, Game8, Mobalytics, GrindNStrat, Oslink 등)를
  종합한 것으로, 정확한 수치(초당 산출량, 정확한 그리드 크기 등)는 검증되지 않았습니다.
- 게임은 계속 패치되고 있으므로(최근 v1.4 「심연으로」에서 기체공업 신설처럼 설비 체계 자체가 바뀌기도 함),
  하드코딩보다는 **사용자가 직접 설비/레시피 데이터를 추가·수정할 수 있는 에디터 화면**을 만드는 걸 추천합니다.
