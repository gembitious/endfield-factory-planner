# 엔드필드 공장 플래너 (Endfield Factory Planner)

명일방주: 엔드필드(Arknights: Endfield)의 통합 공업 시스템(AIC) 설비 배치를 스케치해보는 드래그 앤 드롭 기반 공장 레이아웃 플래너입니다. 게임의 정확한 수치 재현이 아니라, 생산 라인이 병목 없이 맞물리는지 미리 검증하는 **기획 도구**입니다.

Vite + TypeScript(Vanilla, 프레임워크 없음) + HTML5 Canvas로 만들어졌으며, GitHub Pages로 바로 배포됩니다.

## 개발/실행

```bash
npm install
npm run dev      # 개발 서버 (기본 5173 포트)
npm run build    # 타입체크 + dist/ 빌드
npm run preview  # 빌드 결과 미리보기
```

## GitHub Pages 배포

`main` 브랜치에 push하면 [.github/workflows/deploy.yml](.github/workflows/deploy.yml)이 자동으로 빌드해서 GitHub Pages에 배포합니다.

최초 1회 설정: GitHub 저장소 → **Settings → Pages → Build and deployment → Source**를 **GitHub Actions**로 변경.

`vite.config.ts`의 `base: './'` 덕분에 `https://<user>.github.io/<repo>/` 하위 경로에서도 그대로 동작합니다.

## 기능

| 기능 | 설명 |
| --- | --- |
| 배치 | 팔레트에서 드래그 앤 드롭, 또는 클릭 후 캔버스 클릭 (Shift+클릭 = 연속 배치) |
| 회전 | `R` 키 또는 회전 버튼 (90° 단위, 겹치면 회전 불가) |
| 충돌 방지 | 겹치는 칸에는 배치/이동/회전 불가 |
| 연결 | 포트 클릭 → **빈 칸을 클릭할 때마다 경유지 추가** → 반대 포트 클릭으로 완성. 그리는 중 실시간 경로 미리보기, `Backspace`로 경유지 되돌리기. 경유지 없이 포트→포트 클릭 시 자동 최단 경로. 리소스·운송 종류(벨트↔파이프)가 다르면 연결 거부 |
| 칸 점유 라우팅 | 벨트/파이프는 실제로 그리드 칸을 점유합니다. 경로는 클릭한 경유지를 순서대로 지나며, 구간 사이는 설비·같은 종류의 라인을 피해 이어집니다. 라인이 깔린 칸에는 설비를 배치할 수 없고, 경로가 막히면 연결이 거부됩니다 |
| 교차 브리지 | 같은 종류의 라인이 직선×직선으로 수직 교차하면 교차 칸에 브리지가 자동 표시됩니다 (모서리 칸에서는 교차 불가 — 실게임 규칙). 벨트와 파이프는 서로 겹칠 수 있습니다 |
| 벨트/파이프 | 원형 포트·회색 굵은 선 = 벨트(30/분), 마름모 포트·청록 선 = 파이프(120/분). 포트는 설비별 지정 위치(side/pos)에 고정 가능 |
| 썸네일 | 설비별 실제 게임 아이콘 표시 (`public/icons/`, 없으면 이모지 폴백) |
| 저장 | localStorage 자동 저장 + JSON export/import |
| 공유 | "공유 URL 복사" — 레이아웃이 URL 해시(`#L=...`)에 인코딩됨 |
| 전력 커버리지 | 프로토콜 코어/중계기/전력 공급기 배치 시 범위 원 표시. 코어에 연쇄되지 않은 중계기는 점선(비활성). 범위 밖 설비는 빨간 테두리 + ⚡ 경고 |
| 병목 검증 | 연결된 라인의 공급 속도 < 수요 속도이면 해당 연결이 주황색 ⚠️ 하이라이트 |
| 복수 레시피 | 정련로(5종)·분쇄기(6종)·포장기(4종) 등 13개 설비는 정보 패널에서 레시피 선택 가능. 선택한 레시피의 자원이 실측 포트 슬롯(벨트=고체, 파이프=액체/기체)에 자동 배정되고, 연결 검증·병목 계산·저장/공유에 모두 반영 |
| 설비 에디터 | 🛠 설비 편집 — 앱 안에서 설비 추가·수정·삭제·복제. 수정본은 localStorage에 저장되고 JSON으로 내보내 `public/data/facilities.json`을 교체하면 저장소에 반영 (레시피는 JSON 직접 편집) |
| 다크모드 | 우상단 🌙/☀️ 토글 |

### 조작법

- 휠: 줌 / 우클릭·휠클릭 드래그: 화면 이동
- 설비 클릭: 선택(정보 패널 표시) / 드래그: 이동
- `R`: 회전, `Del`: 삭제, `Esc`: 취소/선택 해제

## 데이터 출처와 정확도

- **명칭**: 한국어 정식 서비스 기준 (나무위키 「명일방주: 엔드필드/설비」 대조)
- **footprint·포트 개수/위치/종류(벨트·파이프)·전력 소비량**: [EnKAD](https://enkad.enka.network/)(enka.network의 Endfield 팩토리 플래너)에 포함된 게임 클라이언트 추출 데이터 기준 **실측값** (47종 반영). 예: 프로토콜 코어 9×9에 입력 14·출력 6, 정련로 벨트 입력 3 + 파이프 입력 1, 정제기 파이프 2:2
- **분당 생산 속도(rate)**: 공식 미공개라 커뮤니티 추정치 (표준 라인 30/분 기준)
- 같은 리소스가 여러 포트에 배정된 경우 = 병렬 레인. 생산/수요는 설비 단위로 1회 계산되고 흐름은 연결된 레인에 분배됩니다
- 기체 공업 4종(v1.4 신규)은 아직 추출 데이터에 없어 추정치 유지

## 데이터 수정 (`public/data/facilities.json`)

명칭은 정식 번역을 따르지만, **생산 속도(개/분)·footprint·전력 수치는 공식 미공개라 대략치**입니다. 패치나 실측에 맞게 직접 수정하세요. 표준 라인 = 컨베이어 벨트 최대 운송량인 2초당 1개 = **30/분** 기준입니다 (파이프는 1초당 2개 = 120/분).

```jsonc
{
  "metersPerCell": 4,          // 1 그리드 셀 = 4m (전력 범위 원 크기에 사용)
  "categories": { "자원 채집": { "color": "#d97706" }, ... },
  "facilities": [
    {
      "id": "crusher",          // 고유 id (저장된 레이아웃이 참조하므로 변경 주의)
      "name": "분쇄기",
      "category": "기초 생산",   // categories 키 중 하나
      "icon": "🔨",             // 이미지가 없을 때의 이모지 폴백
      "image": "icons/crusher.png", // 썸네일 (public/ 기준 상대 경로, 선택)
      "footprint": { "w": 3, "h": 3 },
      "powerDraw": 5,           // 전력 소비량 (0 = 무전력)
      "inputs":  [{ "resource": "정련 오리지늄", "rate": 30 }],
      "outputs": [{ "resource": "오리지늄 가루", "rate": 30 }],
      "note": "정보 패널에 표시되는 설명"
    }
  ]
}
```

포트(입력/출력) 항목의 선택 필드:

- `"transport": "pipe"` — 파이프 포트 (생략 = 벨트). 벨트 포트끼리, 파이프 포트끼리만 연결 가능
- `"side": "N"` + `"pos": 1` — 포트를 특정 변(N/E/S/W)의 특정 칸(0부터)에 고정.
  생략하면 입력=서쪽, 출력=동쪽에 자동 분배. 회전 시 포트도 함께 회전

예: 반응기의 청정수 입력을 북쪽 변 가운데 파이프 포트로 —
`{ "resource": "청정수", "rate": 30, "transport": "pipe", "side": "N", "pos": 1 }`

특수 필드:

- `"powerSource": true` — 프로토콜 코어처럼 전력의 원천이 되는 설비 (전력 체인의 시작점)
- `"powerRange": 80` — 전력 공급 범위(미터). 중계기는 활성 전력원 범위 안에 있어야 연쇄 활성화됨
- `"passthrough": true` — 분류기/합류기처럼 들어온 리소스를 그대로 흘려보내는 물류 설비 (`"resource": "any"` 포트와 함께 사용)
- `"maxPerBase": 12` — 정보 표시용 (천화로 등)

컨베이어 벨트·파이프 자체는 팔레트에 없습니다 — 두 설비의 포트를 잇는 **연결선**이 벨트/파이프입니다. 교차가 필요하면 물류 브리지 설비를 사용하세요.

## 레이아웃 JSON 형식

```json
{
  "v": 1,
  "modules": [{ "id": 1, "typeId": "crusher", "x": 0, "y": 0, "rot": 90 }],
  "connections": [{
    "id": 5, "fromModuleId": 1, "fromPort": "out:0", "toModuleId": 2, "toPort": "in:0",
    "waypoints": [{ "x": 4, "y": -2 }]
  }]
}
```

포트 키는 `in:<index>` / `out:<index>`로, `facilities.json`의 inputs/outputs 배열 순서를 가리킵니다.

## 이미지 저작권 고지

`public/icons/`의 설비 아이콘은 게임 **명일방주: 엔드필드**의 인게임 에셋으로, 저작권은 **ⓒ Hypergryph / Gryphline**에 있습니다. 이미지는 [Endfield Talos Wiki](https://endfield.wiki.gg/wiki/Category:Facility_icons)에서 가져왔으며, 이 프로젝트는 수익을 목적으로 하지 않는 비공식 팬 도구입니다. 권리자의 요청이 있을 경우 해당 이미지는 즉시 제거됩니다.
